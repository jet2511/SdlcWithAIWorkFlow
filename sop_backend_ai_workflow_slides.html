Phase 2: TDD & Defensive Execution

Lập Trình Phòng Thủ & Test-Driven Development

1. Chu Trình Red-Green-Refactor
• Red: Viết Unit Test (JUnit 5 + Mockito) chứng minh chưa có tính năng.
• Green: Triển khai logic nghiệp vụ vừa đủ để Test pass.
• Refactor: Tối ưu code mà không làm gãy test suite.
2. Chống N+1 Query (Phòng Thủ)
Sử dụng @EntityGraph(attributePaths = {"user", "action"}) hoặc JOIN FETCH trong JPA Repository để ngăn nổ hàng ngàn query con.
3. Transaction Boundaries & Logging
• @Transactional(readOnly = true) cho query tra cứu.
• Log chuẩn SLF4J: logger.info("Exporting audit log user={}", userId).
🧪 Implementation Code Snippet (ods-service)
// 1. TDD Test Case (JUnit 5 + Mockito) @Test void shouldFilterAuditLogsByDateRange() { Pageable page = PageRequest.of(0, 20); Page<AuditLog> result = service.search(request, page); assertEquals(1, result.getTotalElements()); } // 2. Repository Anti N+1 (@EntityGraph) @EntityGraph(attributePaths = {"user", "targetResource"}) Page<AuditLog> findAllByFilter(AuditFilter filter, Pageable p); // 3. Structured Logging (No String concat) log.info("Processing audit log search criteria={}", filter);
